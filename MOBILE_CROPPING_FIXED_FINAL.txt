THE SOLUTION THAT WORKED:
  Instead of using flexible padding, used FIXED HEIGHT CALCULATIONS
  
  CanvasWrapper mobile:
    height: calc(100vh - 390px)
    max-height: calc(100vh - 390px)
  
  Canvas:
    max-height: calc(100vh - 400px)
  
  This explicitly reserves space for:
    • Header + Search: ~140px
    • Breadcrumbs: ~50px
    • Color Palette: ~90px
    • Toolbar: ~90px
    • Margins: ~20px
    = Total: ~390px

WHAT DIDN'T WORK:
  ❌ Flexible padding (200px, 240px, 280px, 320px, 380px)
  ❌ Scrollable containers
  ❌ Flexible heights with flex: 1
  
WHY FIXED HEIGHT WORKS:
  ✅ Predictable space allocation
  ✅ No guess-work about available space
  ✅ Image scales to fit exactly
  ✅ All controls always visible

After multiple attempts with flexible padding (200px → 380px), 
the solution was to use FIXED HEIGHT CALCULATIONS that explicitly
subtract all UI elements from viewport height.

This ensures:
  • Full image always visible
  • Consistent behavior across devices
  • No cropping
  • All controls accessible
